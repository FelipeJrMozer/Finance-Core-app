# Fase 9 — UX polish

## Contexto

Pequenos detalhes que diferenciam um app amador de um produto premium. Web tem skeletons, sonner, animações de página, hover states. Mobile precisa do equivalente para garantir a mesma "vibe".

## Tarefas

### 9.1 Skeleton loaders consistentes

Criar `components/ui/Skeleton.tsx`:

```tsx
import { useEffect, useRef } from 'react';
import { Animated, View, StyleSheet } from 'react-native';
import { useTheme } from '@/context/ThemeContext';

export function Skeleton({ width, height, borderRadius = 6, style }) {
  const opacity = useRef(new Animated.Value(0.3)).current;
  const { colors } = useTheme();
  useEffect(() => {
    Animated.loop(
      Animated.sequence([
        Animated.timing(opacity, { toValue: 0.7, duration: 800, useNativeDriver: true }),
        Animated.timing(opacity, { toValue: 0.3, duration: 800, useNativeDriver: true }),
      ])
    ).start();
  }, []);
  return (
    <Animated.View style={[
      { width, height, borderRadius, backgroundColor: colors.surfaceElevated, opacity },
      style
    ]} />
  );
}
```

Aplicar em:
- Dashboard: 4 skeletons de SummaryCard, 1 skeleton 200px alto para chart, 4 skeletons de TransactionItem
- Lista de transações: 8 itens fantasmas
- Lista de investimentos: 6 itens fantasmas
- Detalhe de cartão (carregando fatura): linha fantasma do hero + 5 itens
- Reports: skeleton para cada gráfico

Substituir `<ActivityIndicator>` por `Skeleton` onde fizer sentido.

### 9.2 Toast/Snackbar consistente

Criar `components/ui/Toast.tsx` baseado em Reanimated (slide-in do topo). API:

```ts
import { toast } from '@/components/ui/Toast';

toast.success('Lançamento criado');
toast.error('Falha ao salvar');
toast.info('Sincronizando…');
```

- Posicionar no topo, abaixo do safe area, com auto-dismiss em 3s.
- Cor de fundo conforme tipo (`colors.success / .destructive / .info`).
- Swipe-to-dismiss para a direita.
- Substituir todos os `notifyError` (`utils/notify.ts`) por `toast.error`.

### 9.3 Empty states refinados

`components/EmptyState.tsx` já existe. Garantir que **toda** tela vazia tenha:
- Ícone Lucide-equivalent grande (96×96, com `colors.textTertiary`)
- Título 18px medium
- Descrição 14px regular `textSecondary`, max 2 linhas
- CTA primário (PressableScale) levando à ação principal

Telas que precisam revisar empty state:
- Lançamentos (sem nenhum): "Sua jornada começa aqui — adicione seu primeiro lançamento"
- Cartões (sem nenhum): "Cadastre seu primeiro cartão para acompanhar a fatura"
- Investimentos: "Comece sua carteira" + CTA "+ Adicionar ativo"
- Metas: "Defina seu primeiro objetivo"
- Alertas: "Nenhum alerta personalizado ainda"
- Bills: "Sem contas a pagar nos próximos 30 dias 🎉"
- Dividendos: "Nenhum dividendo registrado"

- `(tabs)/_layout.tsx`: transição padrão fade
- Modais (transaction/add, account/add, etc.): `presentation: 'modal'` + slide-up

### 9.5 Pull-to-refresh em todas as listas/dashboards

### 9.6 Haptics em ações críticas

- `Haptics.impactAsync('light')` em todos os taps de itens de lista
- `Haptics.notificationAsync('success')` ao salvar transação/conta/etc.
- `Haptics.notificationAsync('error')` ao mostrar toast de erro
- `Haptics.impactAsync('medium')` em swipe-actions (deletar, arquivar)

Todas envoltas em try/catch (haptics falham em emuladores).

### 9.7 Modal "Adicionar rápido" (FAB melhorado)

- Tap → bottom sheet com 4 ações grandes: "💸 Despesa", "💰 Receita", "🔁 Transferência", "📷 Escanear Pix"
- Cada ação abre o fluxo correspondente já pré-preenchido com a última conta usada (`quickAdd:lastAccountId`)
- Long-press no FAB → ação default: criar despesa

### 9.8 Pull-down dismiss em modais

Tela `transaction/add`, `account/add`, `card/add`, etc.: usar `presentation: 'modal'` do expo-router e permitir swipe-down dismiss (nativo no iOS).

### 9.9 Confirmações destrutivas com `ActionSheet`

```ts

```

Usar em: deletar transação, deletar conta, deletar cartão, deletar meta, deletar categoria, excluir conta de usuário (LGPD).

### 9.10 Estado de "sem internet" mais explícito

Banner já existe (`OfflineBanner`). Adicionar:

- Mutations enquanto offline → enfileirar em `services/offlineQueue.ts` com `AsyncStorage` e replay quando voltar

### 9.11 Acessibilidade (a11y)

- Adicionar `accessibilityLabel` em todo botão e Pressable
- `accessibilityRole="button" / "link" / "header"`
- `accessibilityState={{ disabled, selected }}` onde aplicável
- Garantir contraste mínimo 4.5:1 em todo texto (verificar com tooling tipo `react-native-color-contrast`)
- `accessibilityValue={{ now, min, max }}` em Health Score gauge e barras de progresso
