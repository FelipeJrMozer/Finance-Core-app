# Fase 8 — Performance e arquitetura de dados

## Contexto

`FinanceContext.loadAll()` faz 10 fetches paralelos sem paginação na carga inicial; com 1000+ transações o app trava 3–5s. Listas usam `ScrollView`/`FlatList` simples sem virtualização. Não há cache persistente (perde dados ao reabrir offline).

## Tarefas

### 8.1 Adotar React Query como camada de cache

O pacote `@workspace/api-client-react` já existe (gerado pelo Orval). Mesmo que o OpenAPI atual cubra só `/healthz`, a infra de QueryClient pode ser usada.

- Em `app/_layout.tsx`, montar `<QueryClientProvider client={queryClient}>` na raiz.
- Configurar:
  ```ts
  defaultOptions: {
    queries: {
      staleTime: 60_000,
      gcTime: 5 * 60_000,
      refetchOnWindowFocus: false,
      retry: (count, err) => err.status >= 500 && count < 2,
    }
  }
  ```
- Persistência: usar `@tanstack/react-query-persist-client` com `AsyncStorage` para cachear queries entre sessões. Limite 4MB. Versionar via `buster: appVersion`.

### 8.2 Migrar fetches manuais do FinanceContext para hooks `useQuery`

Hoje `loadAll` faz `apiGet` direto. Refatorar gradualmente:

- `useTransactions(params)` → `queryKey: ['/api/transactions', walletId, params]`
- `useAccounts()`, `useCards()`, `useInvestments()`, `useGoals()`, `useBudgets()`, `useCategories()`, `useTags()`, `useNotifications()`, `useSettings()`
- `useMutation` para todos os adds/updates/deletes, com `onSuccess: () => queryClient.invalidateQueries(...)`.
- Manter `FinanceContext` como **fachada agregadora** (computed values: `totalBalance`, `monthlyIncome`, etc.) consumindo as queries — não mais fonte primária.

### 8.3 Paginação infinita em `(tabs)/transactions.tsx`

- Usar `useInfiniteQuery` com `pageSize=100`.
- `getNextPageParam: (last) => last.length === 100 ? offset + 100 : undefined`.
- `FlashList` (instalar `@shopify/flash-list`) ao invés de `FlatList` — ganho de 5–10x em scroll.
- `onEndReached={fetchNextPage}` + `onEndReachedThreshold={0.5}`.
- Skeleton placeholder com 6 itens fantasmas no primeiro carregamento.

### 8.4 Listas longas: Investments, Goals, Cards

Substituir `FlatList` por `FlashList` em:
- `(tabs)/investments.tsx`
- `(more)/goals.tsx`
- `(more)/cards.tsx`
- `(more)/accounts.tsx`
- `(more)/notifications.tsx`
- `(more)/pending-transactions.tsx`
- `(more)/bills.tsx`

### 8.5 Consumo do `dashboard-summary` agregado

(Iniciado na Fase 1, formalizar aqui.) Em `(tabs)/index.tsx`:
- `useQuery({ queryKey: ['dashboard-summary', walletId], queryFn: getDashboardSummary })`
- Em mutations relevantes: `queryClient.invalidateQueries({ queryKey: ['dashboard-summary'] })` + `POST /api/mobile/dashboard-summary/invalidate`.

### 8.6 Pre-fetch ao logar

Em `AuthContext.login`, após token salvo:
```ts
await Promise.all([
  queryClient.prefetchQuery({ queryKey: ['/api/accounts'], queryFn: ... }),
  queryClient.prefetchQuery({ queryKey: ['/api/transactions', null, {limit:100, offset:0}], queryFn: ... }),
  queryClient.prefetchQuery({ queryKey: ['dashboard-summary'], queryFn: ... }),
]);
```

Isso elimina spinner do dashboard logo após login.

### 8.7 Imagens otimizadas

- Substituir `<Image>` por `<expo-image>` (já instalado) com `cachePolicy="memory-disk"` em todas as fotos de avatar/logo.
- Comprimir assets em `assets/` para WebP onde possível.

### 8.8 Reduzir re-renders no FinanceContext

Hoje o context tem ~20 valores. Cada `setState` re-renderiza toda a árvore. Aplicar:

- Dividir em 3 contexts: `DataContext` (raw data), `MutationContext` (handlers), `ComputedContext` (derived values com `useMemo`).
- OU manter como está mas envolver tudo num `useMemo` no value:
  ```ts
  const value = useMemo(() => ({...}), [transactions, accounts, ..., handlers]);
  ```
- Adicionar `React.memo()` em `TransactionItem`, `SummaryCard`, `BudgetProgress`, `WalletIcon` etc.
- Trocar inline object styles por `StyleSheet.create()` em todos os componentes principais.

### 8.9 Bundle size: code splitting de simuladores

Os 20 simuladores carregam mesmo se nunca usados. Usar `React.lazy` dentro de `app/(more)/simulators/[id].tsx` (rota dinâmica que importa cada arquivo de simulador via dynamic import).

### 8.10 Telemetria de performance

Instalar `expo-application` e medir:
- Tempo do `AuthContext` resolver `isAuthenticated`
- Tempo do `loadAll` completo
- TTI do dashboard

Logar via `services/insights.ts` (criar `POST /api/mobile/perf-event { type, durationMs, version }` se backend permitir).

## Definition of Done

- [ ] Pull-to-refresh em todas as telas principais funciona via React Query.
- [ ] Cache persistido: ao abrir app sem internet, dados aparecem (com banner offline).
- [ ] Lista de lançamentos com 5.000+ itens scrolla a 60fps.
- [ ] Dashboard mostra dados em <500ms após login (pre-fetch).
- [ ] Bundle JS principal não inclui simuladores raramente usados.
- [ ] Re-renders reduzidos: profiler mostra <8 components atualizando ao adicionar uma transação.
