# Fase 4 — Imposto de Renda, DARF e MEI completo

## Contexto

O sistema web tem suite completa de IRPF (geração de arquivo importável da RFB, guia PDF, otimizador, calendário fiscal). O mobile não consome **nenhum endpoint `/api/tax/*`**. Esta é a feature mais diferenciada do web — sem ela o mobile fica com cara de orçamento doméstico.

A parte PJ/MEI do mobile já existe mas é simplificada (só DAS local) — vamos integrar com os endpoints reais e adicionar DASN-SIMEI.

## Endpoints prontos

**Tax (PF + Crypto + Investimentos)**
```
GET   /api/tax/monthly/:year/:month           snapshot mensal de obrigações
GET   /api/tax/journal/:year/:month           livro caixa (operações tributáveis)
GET   /api/tax/annual/:year                   resumo anual
POST  /api/tax/darf/calculate                 calcula DARF a partir de operações
GET   /api/tax/calculations/history           histórico de DARFs gerados
PATCH /api/tax/calculations/:id/mark-paid     marca DARF como pago
GET   /api/tax/irpf/export/:year              gera arquivo .DEC importável
GET   /api/tax/irpf/download/:year            download direto
GET   /api/tax/irpf/guide/:year               guia passo-a-passo (JSON)
GET   /api/tax/irpf/guide/:year/pdf           guia em PDF
POST  /api/tax/optimizer                      sugere movimentos para reduzir IR
GET   /api/tax/calendar/:year                 todas as datas fiscais do ano
GET   /api/tax/accumulated-losses/:year/:month/:assetType/:operationType
GET   /api/tax/crypto/transactions/:year/:month
POST  /api/tax/alerts/configure
GET   /api/tax-incomes                        rendimentos isentos/tributáveis (CRUD)
POST  /api/tax-incomes
PATCH /api/tax-incomes/:id
DELETE /api/tax-incomes/:id
GET   /api/tax-deductions                     deduções dedutíveis (CRUD)
POST  /api/tax-deductions
PATCH /api/tax-deductions/:id
DELETE /api/tax-deductions/:id
```

**PJ / MEI**
```
GET   /api/pj/accounts                contas PJ (CRUD)
GET   /api/pj/tags                    tags PJ
GET   /api/pj/revenues                receitas PJ (não usar mais filtro `[PJ]` em description)
GET   /api/pj/expenses                despesas PJ
GET   /api/pj/das                     DAS pagos (CRUD)
```
*(Verificar se há `/api/pj/dasn-simei` no backend; se não, criar — escopo backend.)*

## Tarefas

### 4.1 Painel IRPF — `(more)/taxes.tsx`

Card grande hero:
- "Você precisa entregar IRPF? **SIM/NÃO/Talvez**" — calculado a partir das regras (rendimentos > teto, posse > teto, vendas > teto)
- Status do ano corrente: "DARF deste mês: R$ X" + botão "Pagar"

Grid de cards:
- **Próximo DARF**: valor + vencimento + botão "Calcular agora" → `POST /api/tax/darf/calculate`
- **Resumo mensal**: ganhos tributáveis no mês, prejuízos acumulados, IR devido (vem de `/api/tax/monthly/:year/:month`)
- **Histórico de DARFs**: lista paginada de `/api/tax/calculations/history` com badge Pago/Pendente/Atrasado, ação "Marcar como pago"
- **Calendário fiscal do ano**: chips por mês com obrigações daquele mês (`/api/tax/calendar/:year`)
- **Tipo de operação**: tabs Ações / FIIs / ETFs / Cripto — para cada uma exibir prejuízos acumulados via `/api/tax/accumulated-losses/...`

### 4.2 Geração e download de IRPF — `(more)/irpf-export.tsx`

- Seletor de ano fiscal (ex.: 2024, 2025)
- Botão "Gerar guia passo-a-passo" → `GET /api/tax/irpf/guide/:year` + `GET /api/tax/irpf/guide/:year/pdf` → abrir PDF com `expo-sharing`
- Botão "Baixar arquivo importável (.DEC)" → `GET /api/tax/irpf/download/:year` → salvar via `expo-file-system` e abrir `Sharing` para o usuário enviar para o computador
- Card "Otimizador IR": input ganho previsto, retorna sugestões via `POST /api/tax/optimizer { plannedSale: X }`

### 4.3 Rendimentos e Deduções — `(more)/tax-incomes.tsx` e `(more)/tax-deductions.tsx`

- CRUD de rendimentos isentos/tributáveis e deduções (saúde, educação, dependentes, previdência)
- Cada item: descrição, valor, tipo (rendimento isento/13º salário/aluguel/dividendo isento/etc.), CPF/CNPJ da fonte pagadora
- Lista filtrável por ano

### 4.4 DARF como tela dedicada — `(more)/darf.tsx`

- Lista de DARFs (`/api/tax/calculations/history`)
- Cada item: mês de competência, valor, código de receita, vencimento, status
- Tap → tela detalhe com:
  - Linha digitável (com botão "Copiar")
  - QR Code Pix (se backend gerar)
  - Botão "Marcar como pago" → `PATCH /api/tax/calculations/:id/mark-paid { paidDate, paidVia }`
  - Botão "Compartilhar" → gera PDF e abre share sheet

### 4.5 PJ/MEI: migrar de filtros locais para endpoints reais

**Hoje** (problema): `pj/receitas.tsx`, `pj/despesas.tsx` filtram `transactions` onde description contém `[PJ]`. Isso é um workaround.

**Correção**:
- Criar `services/pj.ts` consumindo `/api/pj/revenues`, `/api/pj/expenses`, `/api/pj/accounts`, `/api/pj/tags`, `/api/pj/das`.
- `pj/receitas.tsx` e `pj/despesas.tsx`: trocar fonte de dados pelo serviço novo. Manter `MonthNavigator`.
- `pj/index.tsx`: dashboard agora consome agregados reais (faturamento mês, faturamento ano, % do limite MEI R$81k, DAS próximo).
- `pj/das.tsx`: `GET /api/pj/das` lista guias geradas; "Marcar como pago" → `PATCH /api/pj/das/:id { paidDate }`.

### 4.6 DASN-SIMEI — `(more)/pj/dasn-simei.tsx`

- Card "Status DASN do ano anterior": Pendente / Entregue
- Resumo do faturamento agregado por mês do ano fiscal (BarChart 12m)
- Botão "Gerar planilha DASN" → endpoint backend (criar se não existir): `GET /api/pj/dasn-simei/:year` retornando JSON com 12 meses de receita bruta + despesas dedutíveis (se aplicável)
- Botão "Abrir Portal MEI" → `WebBrowser.openBrowserAsync('https://www8.receita.fazenda.gov.br/SimplesNacional/...')`
- Calendário fiscal MEI (último dia útil de maio para entrega da DASN do ano anterior)

### 4.7 Alertas de obrigações fiscais

- Em `(more)/custom-alerts.tsx` (Fase 10 vai migrar para `/api/alert-rules`), pré-popular 3 alertas:
  - "Faturamento MEI passou de 80% do limite" (R$ 64.800)
  - "DARF de IR vence em 5 dias"
  - "DAS do MEI vence dia 20"
- Configurar via `POST /api/tax/alerts/configure { alertType, threshold, channel }`
- Notificação via `expo-notifications` no dispositivo + email se `settings.emailNotifications`.

## Definition of Done

- [ ] Tela de IRPF mostra status de obrigação anual e DARF do mês atual.
- [ ] Botão "Baixar IRPF importável" gera o arquivo `.DEC` e abre o share sheet.
- [ ] Histórico de DARFs lista as mesmas linhas que aparecem no web.
- [ ] PJ/MEI agora puxa receitas/despesas/DAS dos endpoints `/api/pj/*` (verificar requests no Network).
- [ ] DASN-SIMEI mostra resumo do ano e link para portal.
- [ ] 3 alertas fiscais ficam pré-configurados ao habilitar PJ.
- [ ] Cálculos de DARF e prejuízo acumulado batem com web.
