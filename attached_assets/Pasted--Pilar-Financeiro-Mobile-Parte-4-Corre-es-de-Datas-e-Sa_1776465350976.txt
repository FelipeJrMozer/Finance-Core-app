### PROBLEMA 1 — Data exibida nas transações de cartão mostra vencimento, não a compra
Transações de cartão chegam da API com:
- `t.date` = data de vencimento da fatura (ex: "2026-05-10")
- `t.transactionDate` = data real da compra (ex: "2026-04-20")
A UI da tela de transações, ao exibir a data, deve usar a data da compra.
**Arquivos a corrigir:** Qualquer componente/tela que renderiza a data de uma transação no formato de lista.
Localizar onde `item.date` ou `t.date` é formatado para exibição e trocar por:
```typescript
// ANTES:
const displayDate = t.date;
// DEPOIS:
const displayDate = t.transactionDate ?? t.date;

Também aplicar a mesma correção no agrupamento da lista por data (quando as transações
são agrupadas por dia para renderização):

// ANTES (agrupamento):
const groupKey = t.date.substring(0, 10);
// DEPOIS:
const groupKey = (t.transactionDate ?? t.date).substring(0, 10);

PROBLEMA 3 — Fatura do cartão inclui lançamentos pendentes sem indicação visual
Arquivo: artifacts/finance-core/context/FinanceContext.tsx

Localizar computedCreditCards (linha ~681). Atualmente calcula used incluindo
transações pendentes (isPaid === false). Separar em dois valores:

// ANTES:
const computedCreditCards = useMemo(() => {
  const now = new Date();
  return creditCards.map((card) => {
    const openInvoiceMonth = getCurrentInvoiceMonth(card.closingDay, now);
    const { start, end } = getBillingPeriod(card.closingDay, openInvoiceMonth);
    const used = transactions
      .filter((t) => {
        if (t.creditCardId !== card.id) return false;
        if (t.type !== 'expense') return false;
        if (isInvoicePayment(t)) return false;
        const purchaseDate = t.transactionDate ?? t.date;
        return purchaseDate >= start && purchaseDate <= end;
      })
      .reduce((s, t) => s + t.amount, 0);
    return { ...card, used };
  });
}, [creditCards, transactions]);
// DEPOIS — separar posted (confirmado) de pending (pendente):
const computedCreditCards = useMemo(() => {
  const now = new Date();
  return creditCards.map((card) => {
    const openInvoiceMonth = getCurrentInvoiceMonth(card.closingDay, now);
    const { start, end } = getBillingPeriod(card.closingDay, openInvoiceMonth);
    const billingTx = transactions.filter((t) => {
      if (t.creditCardId !== card.id) return false;
      if (t.type !== 'expense') return false;
      if (isInvoicePayment(t)) return false;
      const purchaseDate = t.transactionDate ?? t.date;
      return purchaseDate >= start && purchaseDate <= end;
    });
    // Lançamentos confirmados (posted)
    const usedPosted = billingTx
      .filter((t) => t.isPaid !== false)
      .reduce((s, t) => s + t.amount, 0);
    // Lançamentos pendentes (ainda não confirmados)
    const usedPending = billingTx
      .filter((t) => t.isPaid === false)
      .reduce((s, t) => s + t.amount, 0);
    // `used` = total da fatura (posted + pending), para exibição do valor total comprometido
    const used = usedPosted + usedPending;
    return { ...card, used, usedPosted, usedPending };
  });
}, [creditCards, transactions]);

VERIFICAÇÃO APÓS IMPLEMENTAR
Na lista de transações, uma compra feita no dia 20/abril deve aparecer com data 20/abril,
não com a data de vencimento da fatura (ex: 10/maio)
No dashboard, as despesas do mês de abril devem incluir compras feitas em abril
(mesmo que a fatura vença em maio)
Na tela do cartão, quando há lançamentos pendentes, deve aparecer um aviso indicando
o valor ainda não confirmado separado do valor total da fatura
O total card.used não muda — continua sendo a soma de tudo (posted + pending)
